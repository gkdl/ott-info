# 개인정보 처리방침 / 데이터 안전 / 웹 기반 계정 삭제 — 준비 항목

---

## 1. 개인정보 처리방침 (Privacy Policy)

### 필수 포함 항목

| 항목 | 내용 |
|------|------|
| 수집 항목 | 카카오 닉네임, 프로필 이미지 URL, 기기 식별자(Expo push token, 선택) |
| 수집 목적 | 서비스 제공, 커뮤니티 기능(리뷰·즐겨찾기), UGC 안전 관리 |
| 보유 기간 | 회원 탈퇴 시 즉시 삭제 (CASCADE 설계로 DB 전체 삭제) |
| 제3자 제공 | 카카오(OAuth 인증), Supabase(DB·인증 인프라), TMDB(콘텐츠 정보 조회) |
| 이용자 권리 | 열람·수정·삭제 요청 가능 / 앱 내 마이페이지 또는 웹 삭제 경로 제공 |
| 담당자 연락처 | 서비스 운영자 이메일 |

### 공개 URL 요건
- Apple App Store: 앱 등록 정보 "App Privacy" 섹션에 URL 기재
- Google Play: 앱 등록 정보 "개인정보 처리방침 URL" 필드에 기재
- **반드시 앱 심사 전 실제 접근 가능한 공개 URL**이어야 함
- 예: `https://your-app.com/privacy`

## 2. Google Play 데이터 안전 양식 (Data Safety)

Play Console → 앱 콘텐츠 → 데이터 보안 섹션에서 직접 입력.

### 수집·공유 데이터 신고 항목

| 데이터 유형 | 수집 여부 | 공유 여부 | 목적 |
|------------|---------|---------|------|
| 이름(닉네임) | ✅ | ❌ | 앱 기능 |
| 사진 및 동영상(아바타 URL) | ✅ | ❌ | 앱 기능 |
| 앱 활동(리뷰·즐겨찾기) | ✅ | ❌ | 앱 기능 |
| 앱 내 메시지(리뷰 텍스트) | ✅ | ❌ | 앱 기능 |
| 기기 식별자 | 선택적 | ❌ | 분석/푸시(미사용 시 미수집 체크) |

### 보안 관행 항목
- [x] 전송 중 데이터 암호화 (HTTPS/TLS)
- [x] 데이터 삭제 요청 가능 (앱 내 + 웹 경로 제공)

## 4. 웹 기반 계정·데이터 삭제 경로 (Google Play 필수)

Google Play 정책 변경(2023)으로 **앱 외부에서 접근 가능한 계정 삭제 경로** 필수.

### 구현 방법 (Supabase Edge Function 활용)

```
웹페이지: https://your-app.com/account/delete
  → 이메일 입력 → 인증 코드 발송
  → 코드 확인 → "계정 및 모든 데이터 삭제" 버튼
  → Edge Function delete-account 호출 (이메일로 userId 조회 후 삭제)
```

### 최소 구현 대안 (빠른 출시용)
삭제 요청 이메일 안내 페이지도 Google Play 심사 통과 가능:

```html
<!-- https://your-app.com/account/delete -->
<h1>계정 삭제 요청</h1>
<p>계정 삭제를 원하시면 아래 이메일로 요청해주세요.</p>
<p>삭제 요청 이메일: <a href="mailto:delete@your-app.com">delete@your-app.com</a></p>
<p>처리 기간: 영업일 기준 3일 이내</p>
<p>삭제 범위: 계정 정보, 작성 리뷰, 즐겨찾기 등 모든 데이터</p>
```

### Play Console 등록
Play Console → 앱 콘텐츠 → 데이터 삭제 → URL 입력
`https://your-app.com/account/delete`
