## 1. 로컬 빌드 (gradle, 무료 — EAS 크레딧 안 씀)

CNG라 `android/`를 먼저 생성한 뒤 gradle로 빌드한다. **릴리스 서명은 config plugin이 `credentials.json`을 읽어 자동 적용**된다(아래 4번).

```powershell
# 1) android/ 생성 (서명 설정 자동 주입됨)
npx expo prebuild -p android --no-install

# 2) 빌드
cd android
./gradlew bundleRelease        # → AAB (Play 업로드용)
./gradlew assembleRelease      # → APK (직접 설치/테스트용)
./gradlew clean                # 산출물 정리 (용량 회수)
```

산출물 위치:
- AAB: `android/app/build/outputs/bundle/release/app-release.aab`
- APK: `android/app/build/outputs/apk/release/app-release.apk`

> ⚠️ `npx expo prebuild`를 다시 돌려도 `plugins/withAndroidSigning.js`가 릴리스 서명을 재주입하므로 서명이 유지된다.
> ⚠️ 로컬 gradle 빌드도 내부적으로 `expo export:embed`(Expo 번들러)를 호출한다 → `.env` 값이 번들에 인라인됨(7번 참고).

## 2. EAS 빌드 (클라우드 — 무료 크레딧 소진 시 다음 달 리셋)

```powershell
eas build -p android --profile production    # AAB (production 프로필)
eas build -p android --profile preview       # APK (internal 배포)
eas build:list                               # 빌드 목록 + 상태 + 다운로드 URL
eas build:view                               # 최근 빌드 상세
```

- 결과물은 **EAS 서버에 있음** → 빌드 완료 링크 또는 `expo.dev` 대시보드 → Builds → Download.
- production 프로필은 `eas.json`에서 `credentialsSource: "local"` → `credentials.json`의 키로 서명.
- `eas build --local`은 **Windows 미지원** → 로컬은 1번(gradle) 사용.

## 3. Supabase (Edge Functions / DB)

### 수동 배포
```powershell
npx supabase functions deploy tmdb-proxy     # tmdb-proxy 함수 배포
supabase db push                             # DB 마이그레이션 적용
npx supabase functions logs tmdb-proxy       # 함수 로그
```
> `supabase` 전역 CLI 미설치 시 `npx supabase ...` 사용. 로그인: `npx supabase login`, 링크: `npx supabase link`.

### 자동 배포 (권장) — `main` 브랜치 merge 시
`.github/workflows/deploy.yml`이 **push → main** 에서 자동 실행:
1. `supabase db push` (마이그레이션)
2. `supabase functions deploy` (모든 엣지 함수)
3. `eas update --branch production` (앱 JS OTA 업데이트)

→ **프록시 변경은 main에 merge하면 자동 반영**된다(수동 deploy 불필요).
※ 필요한 GitHub Secrets: `SUPABASE_ACCESS_TOKEN`, `SUPABASE_PROJECT_REF`, `SUPABASE_DB_PASSWORD`, `EXPO_TOKEN`.

### 로그 보는 법
- 대시보드 → Edge Functions → `tmdb-proxy` → **Logs**(console 출력 + boot/shutdown) / **Invocations**(요청별 상태코드).
- 프록시는 **에러 시에만** `console.error('[tmdb-proxy] endpoint=... error=...')` 로그 → 성공 요청은 로그 없음(정상).
- 에러 메시지는 HTTP 응답 본문 `{ error }`로도 전달 → 웹 테스트 시 브라우저 Network 탭에서 즉시 확인 가능.
- ⚠️ 재배포해도 CDN이 응답을 **최대 5~10분 캐싱**(`Cache-Control`) → 변경이 바로 안 보이면 캐시 만료 대기.

## 4. 서명 키스토어 관리

### 키 생성 (최초 1회)
```powershell
keytool -genkeypair -v -keystore ott-info-upload.keystore -alias ott-info -keyalg RSA -keysize 2048 -validity 18250
```
> `-validity 18250` = 50년. 키 파일 + 비밀번호는 **반드시 백업**(분실 시 앱 업데이트 영영 불가). git 커밋 금지.

### `credentials.json` (로컬·EAS 공용 서명 설정)
프로젝트 루트 (gitignore됨):
```json
{
  "android": {
    "keystore": {
      "keystorePath": "ott-info-upload.keystore",
      "keystorePassword": "<키스토어 비번>",
      "keyAlias": "ott-info",
      "keyPassword": "<키 비번>"
    }
  }
}
```
- 로컬 gradle: `plugins/withAndroidSigning.js`가 이 파일을 읽어 `android/app/build.gradle`에 release 서명 주입.
- EAS: `eas.json` production의 `credentialsSource: "local"`이 이 파일을 사용.

### 검증 / 지문 확인
```powershell
# 키스토어 비번·alias 확인 + SHA1/SHA256
keytool -list -v -keystore ott-info-upload.keystore -alias ott-info

# 빌드된 AAB가 어떤 키로 서명됐는지 확인 (릴리스 키 SHA1과 일치해야 함)
keytool -printcert -jarfile android/app/build/outputs/bundle/release/app-release.aab
```
- 릴리스 키 SHA1: `A6:AF:CA:87:E6:D2:A5:83:48:9B:66:51:12:01:7B:F8:99:F9:3C:FB`
- 디버그 키 SHA1(`5E:8F:16:...`)로 나오면 **잘못된 빌드** → Play 업로드 불가.

### 카카오 키 해시 (로그인 필수)
```powershell
keytool -exportcert -alias ott-info -keystore ott-info-upload.keystore | openssl sha1 -binary | openssl base64
```
- 업로드 키 해시: `pq/Kh+bSpYNIm2ZREgF7+Jn5PPs=` → 카카오 개발자센터 → 앱 설정 → 플랫폼 → Android → 키 해시 등록.
- ⚠️ **Play 앱 서명**을 쓰면 구글이 재서명 → Play Console의 **앱 서명 키 SHA-1**으로 만든 해시도 **추가 등록** 필요(첫 업로드 후).

## 5. 환경 변수 (개발/운영 분리)

| 파일 | 용도 | 로드 시점 |
|---|---|---|
| `.env` | **운영(prod)** | 항상 (릴리스 빌드 포함) |
| `.env.development` | **개발(dev)** | `expo start`(개발 모드)만 |
| `.env.example` | 템플릿(커밋) | — |

- 릴리스 빌드(`--dev false`)는 `.env.development`를 안 읽음 → 자동 운영 값. **수동 교체 불필요.**
- dev 백엔드가 생기면 `.env.development`의 `EXPO_PUBLIC_SUPABASE_URL`·`ANON_KEY`만 dev 값으로 교체.
- ⚠️ **EAS 빌드는 `.env*`를 못 받음**(gitignore) → EAS는 `eas.json`의 `env` 또는 EAS 대시보드 환경변수로 별도 지정.

## 7. 이번에 겪은 이슈 & 해결 (참고)

| 증상 | 원인 | 해결 |
|---|---|---|
| `EventQueue::UpdateMode` C++ 에러, RN-screens Kotlin 대량 에러 | RN 0.79 ↔ Expo SDK 56 버전 불일치 | `expo install --fix`로 RN 0.85.3 등 정렬 |
| `Unable to resolve react-native-reanimated` | reanimated 미설치 (NativeWind 4 필수) | `expo install react-native-reanimated react-native-worklets` + babel에 `react-native-worklets/plugin` |
| `지정된 경로를 찾을 수 없습니다`(번들 후) | build.gradle의 hermesCommand가 옛 경로(RN 0.85는 `hermes-compiler`로 이동) | build.gradle의 하드코딩 `hermesCommand` 제거(기본값=자동탐지) |
| 로컬 AAB가 디버그 키로 서명됨 | 로컬 gradle은 `credentials.json`을 안 읽음(prebuild 기본=debug 서명) | `plugins/withAndroidSigning.js` config plugin으로 release 서명 주입 |
| 쿠팡플레이/TVING 추천 빈 화면 | 잘못된 TMDB provider ID | 쿠팡 `1881`, TVING `1883`, Prime `119`로 정정 |
| 성인 영화가 카테고리에 노출 | TMDB `adult:false`라 기본 필터 통과 | 프록시 `tmdbFetch`에 `without_keywords`(성인 키워드) + `include_adult=false` |
| 웹에서 로그아웃/탈퇴 버튼 무반응 | `Alert.alert`는 웹 미지원 | `src/lib/dialog.ts` 크로스플랫폼 다이얼로그 |
