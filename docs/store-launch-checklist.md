# 출시 전 Apple App Store 1.2 & Google Play UGC 점검 리스트

> 체크하지 않은 항목이 하나라도 있으면 심사 반려 또는 정책 위반 경고 가능.

---

## ✅ 공통 UGC 필수 항목

### 사전 필터링
- [ ] 클라이언트 1차 욕설·스팸 필터 적용 (`src/lib/contentFilter.ts`)
- [ ] DB 트리거 2차 필터 또는 CHECK 제약 적용 (rating 1~5, comment ≤ 1000자)
- [ ] 리뷰 작성 시 `include_adult=false` 강제 (TMDB 검색에서도 적용 확인)

### 신고 기능
- [ ] 모든 리뷰 카드에 "신고" 버튼 노출 (비로그인 유저에게는 로그인 유도)
- [ ] 신고 사유 선택 UI 구현 (`REPORT_REASONS` 목록)
- [ ] 신고 저장 후 확인 메시지 표시 ("24시간 내 검토" 안내 포함)
- [ ] 자기 리뷰 신고 불가 처리

### 차단 기능
- [ ] 신고와 동시에 즉시 차단 옵션 제공
- [ ] 차단 후 해당 유저의 콘텐츠가 피드에서 즉시 사라짐 확인
- [ ] 자기 자신 차단 불가 처리

### 24시간 내 조치
- [ ] `admin-flow.md` 기반 신고 누적 알림 트리거 배포
- [ ] 운영자 처리 Edge Function (`admin-moderate`) 배포 및 테스트
- [ ] 신고 접수 → 처리 완료까지 24시간 내 SLA 운영 체계 수립

### EULA 동의
- [ ] 최초 가입 시 EULA 모달 표시 (`EulaModal.tsx`)
- [ ] 동의 거부 시 가입 불가 (로그아웃 처리)
- [ ] `profiles.eula_agreed_at` 저장 확인
- [ ] EULA 본문에 무관용 정책(욕설·혐오·스팸 즉시 제재) 명시

## ✅ 보안 아키텍처 최종 확인

- [ ] `EXPO_PUBLIC_` 접두사 변수에 TMDB 키 / service_role 키 없음 확인
  ```bash
  grep -r "EXPO_PUBLIC_" .env* | grep -iE "tmdb|service_role"
  # 결과 없어야 정상
  ```
- [ ] `supabase secrets list`로 TMDB_API_KEY, SUPABASE_SERVICE_ROLE_KEY 등록 확인
- [ ] 모든 테이블 RLS 활성화 확인
  ```sql
  SELECT tablename, rowsecurity
  FROM pg_tables
  WHERE schemaname = 'public';
  -- 모든 테이블 rowsecurity = true 여야 함
  ```
- [ ] `anon` 키로 다른 유저 데이터 직접 조회 불가 확인 (Supabase API 직접 호출 테스트)
- [ ] Edge Function JWT 검증 로직 작동 확인 (토큰 없이 호출 시 401 반환)
- [ ] Rate limit 동작 확인 (60회 초과 시 429 반환)
