# 시크릿 / 자격증명 보관 위치 (값은 절대 여기 적지 말 것)

> ⚠️ 이 파일은 **public repo에 커밋됨** → **실제 값(비밀번호·토큰·키)을 여기 적지 마세요.**
> "어디에 보관돼 있는지"만 기록하는 참고용 문서입니다. 실제 값은 비밀번호 매니저에.

| 자격증명 | 용도 | 실제 값 보관처 |
|---|---|---|
| **Supabase DB 비밀번호** | DataGrip 등 DB 직접 접속, `SUPABASE_DB_PASSWORD`(GitHub Secret) | 비밀번호 매니저 / GitHub Secret / Supabase 대시보드에서 reset |
| **Supabase Access Token** | CLI·GitHub Actions 배포 | 비밀번호 매니저 / GitHub Secret(`SUPABASE_ACCESS_TOKEN`) |
| **Supabase Project Ref** | `SUPABASE_PROJECT_REF` | (비밀 아님) Supabase → Settings → General → Reference ID |
| **Expo Token** | EAS / `EXPO_TOKEN`(GitHub Secret) | 비밀번호 매니저 / GitHub Secret |
| **릴리스 키스토어 + 비번** | 앱 서명 | 안전 백업(분실 시 앱 업데이트 불가) + `credentials.json`(로컬, gitignore) |
| **카카오 키 / REST API 키** | 카카오 로그인 | `.env`(로컬, gitignore) + 카카오 개발자센터 |

## 접속/발급 위치 메모
- **DB 접속(DataGrip)**: 대시보드 → Connect → **Session pooler** (IPv4) / Host `aws-0-<region>.pooler.supabase.com`, User `postgres.<project-ref>`, DB `postgres`, SSL require
- **DB 비번 reset**: Supabase → Settings → Database → Reset database password
- **Access Token 발급**: Supabase → 우상단 아바타 → Account → Access Tokens
- **Expo Token 발급**: expo.dev → 계정 → Settings → Access tokens
- **GitHub Secrets 등록**: repo → Settings → Secrets and variables → Actions

## 원칙
- 실제 값은 **비밀번호 매니저**에 1차 보관, 사용처(GitHub Secret / DataGrip / .env)에 2차 등록.
- **repo 안 어떤 파일에도 평문 비밀값 금지** (이 repo는 public).
- 토큰류(Access/Expo)는 분실해도 **재발급** 가능. 키스토어만 분실 시 치명적 → 별도 백업 필수.
